Equilíbrios do mercado Galanis (2026) via OpenSpiel

Resolvendo as quatro estruturas com CFR+ tabular para separar capability gap de falha de mecanismo.

Maio de 2026 · galanis-market · 3-round, 11-action discretisation, 50 CFR+ iters

A pergunta

Galanis (arXiv:2604.20050) mostra que LLMs trading num prediction market falham em agregar informação conforme a estrutura informacional fica mais complexa. O preço final mediano cai de ≈ 0.98 nas estruturas fáceis para ≈ 0.5 (coin toss) na Very Hard, mesmo com segurança separável e teorema do Ostrovsky 2012 garantindo agregação em algum equilíbrio Bayes-Nash.

A pergunta que não está respondida no paper: os LLMs estão longe do equilíbrio, ou o equilíbrio já não agrega bem? O Ostrovsky 2012 garante que existe um NE que agrega — mas pode haver outros que não. Sem resolver o jogo, não dá pra separar essas duas hipóteses.

O que fizemos

  • Implementamos o jogo Galanis como um pyspiel.Game: 3 jogadores em rodízio sequencial, chance node uniforme sobre 8 estados, partições conforme cada uma das 4 estruturas, LMSR (β = 0.01, grid discretizada de 11 preços-alvo).
  • Resolvemos via CFR+ tabular do OpenSpiel (50 iterações por estrutura, ≈ 2 min cada num laptop).
  • Calculamos NashConv como diagnóstico de convergência e a distribuição de preço final por ω sob a policy média.
  • Comparamos com (a) os benchmarks empíricos do paper e (b) um baseline analítico de melhor-resposta míope Bayes-Nash.

Caveat formal: para jogos não-soma-zero de 3+ jogadores, CFR+ converge para um coarse correlated equilibrium, não Nash. Para este jogo específico esse gap é pequeno: o subsídio LMSR cria uma deviation unilateral positivamente-EV para qualquer perfil não-agregador, eliminando CCEs babbling. NashConv ficou ≤ 2 × 10⁻⁴ em todos os runs.

O que isso significa

Headline: a degradação observada por Galanis (mediana de erro saltando de 0.018 para 0.718 conforme a complexidade aumenta) não está no equilíbrio — ela é um capability gap entre os LLMs do paper e o melhor-resposta racional. O equilíbrio agrega informação em todas as quatro estruturas.

O Galanis interpreta o resultado dele como sugerindo que AI agents podem sofrer das mesmas limitações que humanos quando precisam raciocinar sobre o que os outros sabem. Nosso resultado mostra que isso é uma observação sobre a capability dos LLMs atuais, não sobre uma propriedade do mecanismo. Best-response play em todas as estruturas agrega ao truth, consistente com Ostrovsky 2012.

Implicação prática para mercados de previsão / futarchy operados por agentes LLM: o caminho de melhoria é capability (treinar agentes para fazer inferência Bayesiana correta e atualizar crenças de ordem superior corretamente), não mecanism design — o mecanismo já está fazendo o seu trabalho.

Por que CFR > empírico nas estruturas fáceis

Curiosamente, o CFR mediana LE = 0.087 é maior que o empírico = 0.018 nas estruturas Easy/Medium. Isso é artefato da nossa discretização: a grid de 11 ações tem piso em 0.917, então log error ≥ −log(0.917) ≈ 0.087. Os LLMs operam com preços contínuos e atingem ≈ 0.99. A leitura justa é qualitativa: ambos agregam quase perfeitamente nas estruturas onde dá pra agregar.

Limitações

  • Só 3 rounds. CFR tabular fica intratável a 6/9 rounds (info-state count escala como células × ações^(moves_antes_de_jogar_de_novo); chega em ~1.7 × 10⁹ pra Very Hard em 9 rounds com 11 ações). Próximo passo: MCCFR ou CFR em C++.
  • Discretização de preço. Grid de 11 ações foi escolhida pra rodar em laptop. Grid mais fina (21 ou 99 ações) baixaria o piso de log error mas o resultado qualitativo (todas as estruturas agregam) é o mesmo.
  • CCE ≈ NE, não ≡ NE. Em 3 jogadores soma-geral, CFR+ converge formalmente pra coarse correlated equilibrium. Pra este jogo o gap é negligível (argumento via subsídio LMSR), mas é um caveat formal.

Como reproduzir

git clone <repo>
cd galanis-market
pip install -e .
pytest -q                                # 44 testes
python scripts/solve_all.py \
    --iterations 50 --num-actions 11 \
    --structures t3s111y2,t3s110,t3s111,t3s111o2ye2 \
    --rounds 3
python scripts/write_equilibria_doc.py \
    results/cfr_a11_i50_*.json \
    --output results/equilibria.md

Tempo total: ≈ 8 minutos num laptop CPU.
